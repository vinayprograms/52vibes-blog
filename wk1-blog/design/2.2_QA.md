# QUALITY ASSURANCE

This document specifies quality assurance test procedures for validating that the 52vibes Blog Platform meets all requirements documented in `2.1_REQUIREMENTS.md`. Each specification describes how to verify the associated requirement. Test procedures are designed for execution by a QA testing agent under customer supervision.

---

## Blog Platform Specifications

### Hosting and Deployment

* **BP_QA_001** - Verify blog URL accessibility.
  + Navigate to `https://52vibes.dev/blog` in a browser
  + Confirm the page loads without HTTP errors (200 status)
  + Confirm blog content is displayed (not error page, redirect, or placeholder)

* **BP_QA_002** - Verify Cloudflare Pages hosting.
  + Access Cloudflare dashboard for the 52vibes.dev domain
  + Confirm Pages project is configured and active
  + Confirm deployment source is linked to correct repository

* **BP_QA_003** - Verify automatic deployment trigger.
  + Make a trivial change to a content file in the repository
  + Commit and push to the main branch
  + Observe Cloudflare Pages dashboard for build initiation
  + Confirm build starts within 60 seconds of push

* **BP_QA_004** - Verify deployment completion time.
  + Following BP_QA_003, monitor build progress
  + Record time from push to "deployed" status
  + Confirm total time is under 5 minutes
  + Verify changed content appears on live site

### Performance and Compatibility

* **BP_QA_005** - Verify Lighthouse performance score.
  + Run Lighthouse audit on `/blog/` (blog index)
  + Run Lighthouse audit on a weekly post page
  + Run Lighthouse audit on `/about/`
  + Confirm performance score ≥95 on all tested pages
  + Document scores for each page tested

* **BP_QA_006** - Verify JavaScript-free functionality.
  + Disable JavaScript in browser settings
  + Navigate to blog index; confirm content renders
  + Navigate to a blog post; confirm content renders
  + Click internal navigation links; confirm navigation works
  + Click table of contents links; confirm anchor navigation works
  + Use theme toggle; note if it degrades gracefully

* **BP_QA_007** - Verify minimum viewport support.
  + Set viewport width to 320px; confirm layout does not break
  + Set viewport width to 2560px; confirm layout remains usable
  + Confirm no horizontal scrolling required at any tested width
  + Confirm text remains readable at all widths

* **BP_QA_008** - Verify responsive breakpoints.
  + Test at 1440px (desktop); confirm desktop layout
  + Test at 768px (tablet); confirm adapted layout
  + Test at 375px (mobile); confirm mobile layout
  + Confirm navigation adapts appropriately at each breakpoint
  + Confirm content remains accessible at each breakpoint

* **BP_QA_009** - Verify CLI browser compatibility with lynx.
  + Open the blog: `lynx https://52vibes.dev/blog`
  + Confirm all text content is readable
  + Confirm navigation links are present and functional
  + Navigate to a blog post; confirm content is readable
  + Confirm table of contents links work
  + Navigate to Weekly Index; confirm week listing is readable

* **BP_QA_010** - Verify CLI browser compatibility with w3m.
  + Open the blog: `w3m https://52vibes.dev/blog`
  + Navigate to Weekly Index page (`/blog/weeks/`)
  + Confirm the 52-week table renders correctly and is readable
  + Confirm table cells are properly aligned and navigable
  + Navigate to a blog post; confirm content is readable

* **BP_QA_011** - Verify navigation works in CLI browsers.
  + In lynx or w3m, test all navigation links
  + Confirm bottom navigation bar links are accessible
  + Confirm "Weeks" link navigates to Weekly Index
  + Confirm no navigation relies on JavaScript or hover states

### Visual Design

* **BP_QA_012** - Verify terminal/tmux aesthetic.
  + Observe page layout for top status bar presence
  + Observe page layout for bottom navigation bar presence
  + Observe main content area for bordered frame styling
  + Confirm overall visual matches terminal/tmux design intent

* **BP_QA_013** - Verify CSS border implementation.
  + Open browser developer tools
  + Inspect border elements on content frames
  + Confirm borders use CSS `border` property
  + Search page source for ASCII box-drawing characters (─│┌┐└┘├┤┬┴┼)
  + Confirm no ASCII characters used for structural borders

* **BP_QA_014** - Verify dual theme implementation.
  + Set system preference to dark mode; observe page
  + Set system preference to light mode; observe page
  + Confirm distinct dark and light themes are applied
  + Confirm both themes appear intentionally designed

* **BP_QA_015** - Verify dark theme colors.
  + Activate dark theme
  + Inspect background color of main content area
  + Confirm background is `#1d2021` or approved Gruvbox dark variant
  + Confirm text uses light foreground color

* **BP_QA_016** - Verify light theme colors.
  + Activate light theme
  + Inspect background color of main content area
  + Confirm background is `#fbf1c7` or approved Gruvbox light variant
  + Confirm text uses dark foreground color

* **BP_QA_017** - Verify system preference detection.
  + Set OS to dark mode; load blog fresh (clear cache)
  + Confirm dark theme is applied automatically
  + Set OS to light mode; load blog fresh
  + Confirm light theme is applied automatically

* **BP_QA_018** - Verify theme toggle visibility.
  + Load any blog page
  + Locate theme toggle control visually
  + Confirm toggle is discoverable without scrolling
  + Confirm toggle purpose is clear (icon or label)

* **BP_QA_019** - Verify theme toggle persistence.
  + Enable JavaScript
  + Click theme toggle to change theme
  + Navigate to another page
  + Confirm selected theme persists
  + Close and reopen browser; confirm persistence (if localStorage used)

* **BP_QA_020** - Verify theme toggle graceful degradation.
  + Open page in CLI browser (lynx)
  + Confirm page is readable without theme toggle functionality
  + Confirm no errors or broken elements due to missing theme toggle

* **BP_QA_021** - Verify body text typography.
  + Inspect body paragraph text using developer tools
  + Confirm computed font-family includes "IBM Plex Sans"
  + Confirm font renders correctly (not system fallback)

* **BP_QA_022** - Verify code typography.
  + Navigate to a post with code blocks
  + Inspect code block text using developer tools
  + Confirm computed font-family includes "JetBrains Mono"
  + Confirm monospace rendering is correct

* **BP_QA_023** - Verify font loading.
  + Open Network tab in developer tools
  + Load a blog page
  + Confirm web font files are requested and loaded
  + Confirm no FOUT (Flash of Unstyled Text) or graceful handling

* **SC_QA_005** - Verify CSP header presence.

* **SC_QA_007** - Verify CSP script-src.

  + Confirm no overly permissive wildcards

* **SC_QA_010** - Verify CSP font-src.

* **SC_QA_011** - Verify CSP connect-src.

* **SC_QA_012** - Verify no unsafe-eval.
